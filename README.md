# Hyper-Targeted Search-Distribution Framework

Status: source-wired research reference; not a functioning production web framework
Updated: 2026-07-17

## What exists now

The repository contains a deterministic TypeScript laboratory for:

- HDC/VSA/HRR feature composition and contextual matching;
- corpus-coverage and distinctness experiments;
- typed evidence/content/page IR;
- packed vectors and CSR graph artifacts;
- deterministic neutral HTML, schema, sitemap, and instruction emissions;
- synthetic scale tests through 2,000 page fixtures;
- source scaffolds for Zig/Wasm, Cloudflare, field cohorts, and conversion metrics.

It does **not** yet contain:

- a production UI renderer or application shell;
- a live indexed 200–2,000-page corpus;
- production Cloudflare or Zig/Wasm proof;
- real search-console, ranking, citation, conversion, or revenue evidence;
- production content generated by the agent harness.

Synthetic scale proves that the reference compiler can emit many deterministic fixtures. It does not prove that those fixtures deserve to be pages or will populate real search results.

## Single authority

`site-manifest.yaml` is the unified declarative authority.

It contains:

- vector-space axes and link policy;
- the agent-generation harness contract;
- evidence, claims, information objects, and semantic modules;
- page prototype sets;
- Request Mirror, UI-scaffold, geometry-fixture, and field-candidate profiles;
- publication state.

The old five-slice Request Mirror is no longer a separate architecture. It is a `noindex` profile compiled through the same manifest composer as every future page.

## Primary architecture

```text
research + business truth + demand evidence
-> hyper-aware agent proposal
-> typed feature atoms and prototype sets
-> first-class compiled hyper-vector space
-> coverage/neighbor/graph/UI derivation
-> evidence-bounded semantic modules
-> PageIR and packed information server
-> deterministic page and machine-readable emissions
-> later UI renderer
-> gated field publication and commercial measurement
```

The hyper-vector space is created before page emissions. It feeds:

- page eligibility and corpus coverage;
- nearest-neighbor and internal-link suggestions;
- agent generation context;
- semantic module and design-capability requirements;
- finite variants;
- packed vectors and resolver artifacts;
- the UI scaffolding plan;
- final HTML and instruction projections.
