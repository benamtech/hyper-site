# 16 — Unified Hyper-Vector Manifest, Agent Harness, and UI Readiness

Status: source-wired research authority
Updated: 2026-07-17

## Reality reset

The repository is not yet a functioning production web framework. It currently contains:

- a deterministic TypeScript research package;
- synthetic HRR, benchmark, optimizer, packed-IR, emission, resolver, Zig/Wasm-source, and Cloudflare-source experiments;
- a synthetic scale harness that has compiled up to 2,000 page fixtures locally;
- no production UI renderer, deployed public corpus, indexing evidence, ranking evidence, conversion lift, or Zig/Wasm/Cloudflare acceptance.

The previous `site-manifest.yaml` described a separate five-candidate Request Mirror lab. It referenced content files that were not the source of the newer compiler. That split is removed.

`site-manifest.yaml` is now the single declarative authority compiled by `reference/src/manifest.ts`. The Request Mirror is a non-indexable profile inside the same corpus and vector-space compiler.

## First-class compiled state

The canonical pipeline is:

```text
approved research and company source
-> agent-aware feature atoms
-> page/context prototype sets in the hyper-vector space
-> vector-derived coverage, neighbors, graph edges, and UI requirements
-> evidence-bounded semantic modules
-> SiteSource
-> PageIR + packed vectors + CSR graph
-> deterministic HTML, schema, sitemap, instruction, UI-scaffold, and agent-context emissions
```

HTML is an emission. The vector information space, evidence graph, semantic modules, dependencies, and page geometry are authoritative.

## Agent-first generation boundary

Agents do not directly publish HTML or arbitrary pages. They submit an `AgentPageProposal` containing:

- target feature atoms and prototype region;
- canonical question;
- marginal-coverage hypothesis;
- distinct information object;
- evidence and claim bindings;
- semantic modules;
- graph/link rationale;
- design-capability requirements;
- generation provenance.

Every agent proposal enters as `noindex` at the `research` gate. The deterministic composer recompiles the complete vector space, derives neighbors and UI inputs, validates evidence and references, and emits fixtures. Publication is a separate gate.

## External research floor

### Context-document compatibility paper

Deguang Kong, Daniel Zhou, Zhiheng Huang, and Steph Sigalas, “Personalized Search Via Neural Contextual Semantic Relevance Ranking,” arXiv:2309.05113:

https://arxiv.org/abs/2309.05113

Applicable findings:

- document-to-context compatibility is distinct from ordinary query-to-document relevance;
- lexical and semantic contextual signals can be evaluated together;
- context ablation and out-of-domain evaluation are appropriate controls.

Not established by the paper:

- HDC/VSA or HRR as the representation;
- automatic construction of a public web corpus;
- indexing or ranking lift on Google/Bing;
- zero-volume page strategy;
- agent-generated content quality;
- commercial conversion or revenue lift.

The paper is therefore a required baseline, not validation of this framework.

### HDC/VSA applications survey

Denis Kleyko, Dmitri A. Rachkovskij, Evgeny Osipov, and Abbas Rahimi, “A Survey on Hyperdimensional Computing aka Vector Symbolic Architectures, Part II: Applications, Cognitive Models, and Challenges,” arXiv:2112.15424; ACM Computing Surveys 55(9), DOI 10.1145/3558000:

https://arxiv.org/abs/2112.15424

Applicable findings:

- HDC/VSA supports structured distributed representations and cognitive-computing operations;
- deterministic behavior, associative retrieval, compositional representations, and hardware-oriented implementations are legitimate research directions.

Not established:

- SEO advantage;
- public-web corpus construction;
- page distinctness;
- search distribution;
- UI metaprogramming;
- commercial value.

## Validation vector — unified architecture

Run the same manifest through:

1. manifest parsing and structural validation;
2. feature-atom and prototype compilation;
3. packed-vector parity checks;
4. vector-derived neighbor/link generation;
5. semantic `PageIR` compilation;
6. neutral HTML and instruction emissions;
7. hyper-aware agent-context generation;
8. UI-scaffold-plan generation;
9. source-order determinism tests;
10. agent proposal insertion and complete recompilation.

Measure:

- manifest authority count;
- undeclared axes/values;
- pages without prototypes;
- packed/vector parity;
- duplicate routes and IDs;
- invalid evidence references;
- profile/page inconsistency;
- auto-link determinism;
- UI module/capability coverage;
- agent proposal rejection reasons;
- indexable page count before publication approval;
- byte-identical build and space hashes.

### Pass vector

- Exactly one manifest controls lab, UI, geometry, field-candidate, and future production profiles.
- Every page has at least one typed hyper-vector prototype before modules or HTML are emitted.
- The compiled primary prototype matches the packed page vector within declared tolerance.
- Vector-space state deterministically feeds graph links, agent context, UI scaffold requirements, and page emissions.
- The legacy five-slice lab compiles through the same `SiteSource -> PageIR` path as every future page.
- All current pages remain `noindex` while `production_ready=false`.
- Agent proposals require source provenance, target geometry, information gain, evidence, and noindex research entry.
- The UI pass receives module kinds, layout roles, capabilities, page fixtures, variant axes, and vector-space hash without reading hand-authored HTML.
- CI compiles, tests, emits the unified manifest, and uploads the emitted fixture artifact.

### Fail vector

- The Request Mirror and large-scale compiler remain separate authorities.
- Hyper-vectors are calculated only after page content has already been selected.
- Page geometry does not affect graph edges, UI requirements, agent context, or emissions.
- Agents directly create production HTML or indexable routes.
- Synthetic 2,000-page compilation is described as a functioning production site.
- An indexable page exists while the manifest is research-only.
- UI scaffolding requires AMTECH-specific components inside the compiler core.
- The contextual-ranking paper is cited as proof of HRR, SEO lift, or automatic corpus generation.
- The HDC/VSA survey is cited as proof of ranking or revenue.

## First UI metaprogramming pass

The next pass consumes `deriveUiScaffoldPlan(compileFrameworkManifest(site-manifest.yaml), "ui-scaffold")`.

It must implement a design-system superset for:

- all required semantic module kinds;
- all layout roles;
- responsive and mobile constraints;
- evidence, workflow, comparison, FAQ, conversion, and instruction surfaces;
- visible structured data parity;
- reduced motion and accessibility;
- deterministic renderer fixtures.

The renderer may change visual expression. It may not change page geometry, canonical questions, evidence, information objects, graph edges, profile membership, or publication status.
