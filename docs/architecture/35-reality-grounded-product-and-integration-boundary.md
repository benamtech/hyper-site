# Reality-Grounded Product Boundary

Status: accepted architecture correction  
Updated: 2026-07-18

## Product path

```text
approved source material
-> @amtech/hyper-content
-> portable content and site contracts
-> @amtech/hyper-site
-> deterministic static artifacts
```

## Ownership

Hyper Site owns content-neutral compilation, PageIR, deterministic web artifacts, rendering contracts, browser and accessibility budgets, dependency metadata, and static publication.

Hyper Content owns source and evidence intake, claim and information-object proposals, page selection, structured generation, PCN and ArticleIR acceptance, corpus validation, and portable output for Hyper Site.

## Dependency rule

A dependency may enter an active plan only when a current fixture exposes a concrete requirement, the simplest local implementation fails a measured criterion, the dependency improves a named pass metric, and removal remains practical.

A package is never adopted merely because it is established or mature.

## Current implementation scope

Build package-owned TypeScript source, deterministic schemas, static compilation, validation and rejection, provenance, dependency indexing, local publication, and direct comparison fixtures.

## First task boundary

```text
typed input
-> schema validation
-> deterministic local transition
-> local record
-> result or rejection
```

The first task is side-effect free and limited to requirements demonstrated by the fixture.

## Direct controls

Ontology is compared with typed data. Embeddings are compared with lexical rules. Packed vectors are compared with arrays and maps. Wasm and GPU work are compared with direct implementations. Generated tasks are compared with ordinary forms. Automated page selection is compared with an explicit page list.

## Initial proof

```text
one approved evidence set
-> five justified pages
-> one static deployment
-> one measured maintenance event
```

P1.4 and P1.5 remain first. P1.6 starts locally. New dependencies require measured necessity.
