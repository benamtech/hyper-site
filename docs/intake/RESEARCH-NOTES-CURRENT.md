## Reality correction

The repository previously contained two incompatible stories:

1. `site-manifest.yaml`: a five-candidate, noindex Request Mirror research manifest.
2. `reference/`: a separate synthetic compiler able to emit thousands of fixtures.

Neither was a production web framework. The scale report described a synthetic compiler run, not 2,000 reviewed or deployed SEO pages.

The architecture is now unified: one manifest compiles lab, UI, geometry, and future field profiles through one vector-space and `PageIR` path.

## Research thesis

A page is a typed compatibility region, not merely a document. Its primary coordinate can include:

```text
industry
problem
use case
location
audience
stage
proof requirement
constraint
surface/instruction role
```

The corpus problem is to select and generate a finite page set that maximizes valuable context coverage while preserving distinct information gain, evidence, graph utility, maintainability, and commercial value.

The hyper-vector space must exist before content emission. It supplies context to generation agents and constrains page proposals.

## Agent-first hypothesis

A hyper-aware generation harness may outperform generic article prompting because it supplies:

- declared target region;
- current nearby pages and similarity;
- missing information-object requirement;
- evidence inventory;
- graph/cannibalization constraints;
- design capabilities;
- publication and validation gates.

This remains unvalidated. The next evidence is whether agents using this context produce more distinct, useful, evidence-safe page proposals than ordinary briefs.

## Context-ranking paper applicability

Paper:

https://arxiv.org/abs/2309.05113

It supports the premise that document-context compatibility is a separable ranking problem and that lexical plus semantic context should be tested. It does not prove:

- HDC/VSA superiority;
- public-web SEO lift;
- automatic page generation;
- zero-volume targeting;
- agent-first publishing;
- revenue.

Required benchmark role: strongest contextual-ranking baseline against which HRR/hybrid geometry and corpus construction must be tested.

## HDC/VSA survey applicability

Paper:

https://arxiv.org/abs/2112.15424

It supports HDC/VSA as a broad compositional and cognitive-computing framework. It does not validate SEO. Its relevance is architectural: structured distributed representations, deterministic operations, associative retrieval, compositionality, and possible efficient hardware/runtime implementations.

## Zero-volume target hypothesis

A zero-volume combination may be valuable when it:

- represents a real buyer/operator problem;
- has distinct information gain;
- strengthens a larger entity/task graph;
- earns relevant impressions, citations, internal engagement, or assisted conversions;
- costs little to create and maintain.

Zero reported volume alone is not a pass condition. Large numbers of zero-demand pages can still be waste, crawl burden, or scaled thin content.

## Re-indexing latency correction

A universal 90-day re-indexing law is not accepted. Discovery, recrawl, indexing, canonical reassessment, and ranking changes vary by site, URL, demand, links, quality, and system behavior. Field cohorts must use predeclared observation windows and actual Search Console/Bing evidence rather than a fixed folklore delay.

## Current falsifiable questions

1. Does the unified manifest remain deterministic under source reordering and agent proposal insertion?
2. Does a hyper-aware agent harness produce better page opportunities than conventional briefs?
3. Does vector-derived linking improve graph utility without semantic nonsense?
4. Does the UI scaffold remain satisfiable by a clean design-system superset?
5. Do 20–40 reviewed field pages generate more compatible non-branded discovery and qualified pipeline per page than conventional/graph controls?
6. Does HRR add measurable value beyond lexical, semantic, facets, and graph baselines?
