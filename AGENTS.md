## Scope

This subtree is an agent-first hyper-targeted publishing compiler and vector information server research project. It is not yet a production web framework.

`site-manifest.yaml` is the single declarative corpus authority. The Request Mirror, UI fixtures, geometry fixtures, and future field pages are profiles inside that manifest and must compile through the same path.

## Core invariants

1. Feature atoms and page/context prototype geometry are compiled before content modules, links, UI scaffolds, resolver artifacts, or HTML emissions.
2. HTML, Markdown, sitemaps, packed vectors, and UI plans are generated artifacts, never canonical source truth.
3. Agents propose typed `AgentPageProposal` objects. They do not directly publish HTML, routes, claims, or indexable pages.
4. Every agent proposal enters `noindex` at the `research` publication gate.
5. A proposed page requires a canonical question, target vector region, marginal-coverage hypothesis, distinct information object, evidence bindings, graph rationale, design requirements, and provenance.
6. Claims cannot exceed evidence level.
7. Synthetic dimensions and scale fixtures are research labels, not live visitor inference or proof of SEO value.
8. No raw IP persistence, fingerprinting, identity stitching, private-history ingestion, sensitive-trait inference, or named-person profiling.
9. Crawlers and people receive the same primary topic, facts, price, limitations, proof, and links.
10. Resolver error, low confidence, version mismatch, timeout, or policy failure returns the complete baseline.
11. HRR, graph scoring, ANN, Zig/Wasm, optimized artifacts, generated UI, and agent harnesses must beat explicit simpler baselines before adoption.
12. Classical HRR/VSA is not quantum computing.
13. Synthetic 2,000-page compilation must never be described as 2,000 reviewed, deployed, indexed, or revenue-producing pages.

## Compiler path

```text
approved research/company source
-> unified manifest
-> hyper-vector prototype space
-> vector-derived coverage/neighbors/graph/UI requirements
-> evidence-bounded semantic modules
-> SiteSource
-> PageIR + packed vectors + CSR graph
-> deterministic HTML/schema/sitemap/instruction/UI/agent emissions
-> gated field publication
```

Required properties:

- deterministic builds and source ordering;
- declared axes, stable IDs, versions, and hashes;
- evidence/reference validation;
- profile/page consistency;
- primary-vector and packed-vector parity;
- dependency-bounded invalidation;
- no hand-editing generated artifacts;
- build failure on claim, evidence, geometry, profile, design, route, or dependency contradictions.

## Agent-generated page work

Use `buildHyperAwareAgentContext()` before proposing content. The agent context must include the current vector-space hash, target neighborhoods, nearby pages, information objects, design capabilities, generation rules, and publication boundary.

Insert proposals only through `applyAgentPageProposal()` and recompile the complete manifest. A page is not accepted merely because it compiles.

## UI metaprogramming

The first UI pass consumes the `ui-scaffold` profile through `deriveUiScaffoldPlan()`.

The renderer may derive a clean professional design-system superset, but it must preserve:

- feature atoms and prototype IDs;
- canonical questions and semantic module sequence;
- claims, evidence, and information objects;
- internal graph edges;
- required layout roles and capabilities;
- profile membership, indexability, and publication state;
- metadata, structured data, instruction projections, accessibility, and JS-disabled completeness.

## Field publication

A page may enter a field cohort only after relevance, information gain, evidence, distinctness, UI, accessibility, browser, crawl, canonical, internal-link, and lifecycle-cost gates pass.

Do not use a fixed 90-day folklore rule as acceptance. Record real crawl, index, query, citation, lead, revenue, gross-profit, and lifecycle-return evidence over a predeclared observation window.

## Validation reporting

Every substantive implementation records:

- commands and CI runs;
- exact fixtures, versions, hashes, and profiles;
- correctness, relevance, performance, SEO, accessibility, privacy, and security findings;
- comparisons with rules, facets, lexical/semantic, graph, static, and neutral-renderer controls;
- failures, fallbacks, and validation not run.

Phase status never advances from documentation or synthetic scale alone.
